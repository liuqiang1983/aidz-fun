## 写在前面

产品经理是最适合用 AI 提效的岗位之一。

**为什么？**

- 工作内容高度结构化（需求文档、竞品分析、数据报告）
- 大量重复性工作（整理反馈、写文档、做 PPT）
- 需要快速学习新领域（行业调研、技术理解）
- 沟通成本高（跨部门对齐、向上汇报）

AI 可以帮你：

- ✅ 需求分析：快速整理用户反馈、提取核心需求
- ✅ 竞品调研：自动抓取信息、生成对比报告
- ✅ PRD 写作：生成文档框架、润色语言
- ✅ 数据分析：自动生成报告、发现异常
- ✅ 用户访谈：准备访谈提纲、整理访谈记录
- ✅ 项目管理：生成甘特图、风险清单

**实测效果**：一个 3 年经验的 PM，用 AI 后文档产出效率提升 40%，会议准备时间减少 50%。

## 一、需求分析：从混乱到清晰

### 场景 1：整理用户反馈

**痛点**：
- 用户反馈散落在客服、销售、社群、应用商店等多个渠道
- 格式不统一、表述模糊
- 需要人工逐条阅读、分类、提取

**AI 辅助流程**：

```
原始反馈 → AI 分类 → AI 提取核心需求 → AI 生成需求池
```

**Prompt 模板**：
```
我收集了以下用户反馈，请帮我：
1. 按功能模块分类（如：登录、支付、搜索、个人中心等）
2. 提取每个分类下的核心需求（用用户故事格式）
3. 标注高频需求和紧急需求
4. 生成需求优先级建议（考虑频次、影响面、实现成本）

用户反馈：
[粘贴反馈内容，每条一行]
```

### 场景 2：需求评审准备

**Prompt 模板**：
```
我要做一个 [功能名称] 的需求评审，请帮我准备评审材料：
1. 功能背景（为什么做、解决什么问题）
2. 用户场景（3-5 个典型场景）
3. 核心功能点（按优先级排序）
4. 关键指标（如何衡量成功）
5. 风险点（可能遇到的问题）

背景信息：
- 产品：[XX]
- 目标用户：[XX]
- 当前痛点：[XX]
```

### 场景 3：需求文档生成

**Prompt 模板**：
```
请帮我生成 [功能名称] 的 PRD 文档大纲，包括：
1. 需求背景
2. 用户故事
3. 功能详细说明（前端 + 后端）
4. 交互流程
5. 数据埋点
6. 验收标准
7. 风险评估

功能概述：[XX]
目标用户：[XX]
核心价值：[XX]
```

## 二、竞品调研：从手动到自动化

### 场景 1：竞品信息收集

**Prompt 模板**：
```
请帮我调研 [产品名称] 的以下信息：
1. 产品定位和目标用户
2. 核心功能列表
3. 商业模式
4. 最近一年的重大更新
5. 用户评价（App Store / 应用商店评分趋势）

请用表格形式输出，并标注信息来源。
```

### 场景 2：竞品对比分析

**Prompt 模板**：
```
请对比以下竞品，生成对比报告：

竞品列表：
- 产品 A：[名称]
- 产品 B：[名称]
- 产品 C：[名称]

对比维度：
1. 核心功能
2. 用户规模
3. 商业模式
4. 用户体验
5. 技术架构（如有公开信息）

请输出：
- 对比表格
- 各产品优劣势分析
- 对我们的启示
```

### 场景 3：行业趋势分析

**Prompt 模板**：
```
请帮我分析 [行业/赛道] 的最新趋势：
1. 市场规模和增长趋势
2. 主要玩家及市场份额
3. 用户需求变化
4. 技术创新方向
5. 未来 1-2 年的预测

请引用权威数据来源。
```

## 三、PRD 写作：从空白到完整

### 场景 1：PRD 框架生成

**Prompt 模板**：
```
我要写一个 [功能名称] 的 PRD，请帮我生成完整框架：

产品背景：[XX]
目标用户：[XX]
核心价值：[XX]

请按以下结构生成：
1. 文档信息（版本、作者、日期）
2. 需求背景
3. 用户故事
4. 功能详细说明
   - 前端交互
   - 后端逻辑
   - 数据结构
5. 非功能性需求
   - 性能要求
   - 安全要求
   - 兼容性
6. 数据埋点
7. 验收标准
8. 上线计划
9. 风险评估
```

### 场景 2：用户故事编写

**Prompt 模板**：
```
请帮我将以下需求转化为用户故事格式：

需求描述：[XX]

要求：
1. 使用 "作为...我希望...以便..." 格式
2. 每个用户故事包含验收标准
3. 标注优先级（P0/P1/P2）
```

### 场景 3：交互流程设计

**Prompt 模板**：
```
请帮我设计 [功能名称] 的交互流程：

用户场景：[XX]
前置条件：[XX]

请输出：
1. 主流程（正常路径）
2. 异常流程（错误处理）
3. 边界情况
4. 状态流转图（用 Mermaid 语法）
```

## 九、常见问题

**Q：AI 生成的 PRD 能直接用吗？**
A：不能。AI 生成的是框架和初稿，你需要根据实际情况调整、补充细节、验证可行性。

**Q：竞品调研用 AI 准确吗？**
A：AI 提供的信息可能过时或不准确，重要信息需要人工核实。建议用 AI 快速收集，人工验证关键数据。

**Q：数据分析能完全交给 AI 吗？**
A：不能。AI 可以帮你发现模式、生成报告，但业务洞察需要你的判断。AI 是辅助，不是替代。

**Q：用户访谈用 AI 会不会不专业？**
A：AI 帮你准备提纲、整理记录，但访谈过程必须你自己做。AI 提升效率，不改变本质。

**Q：团队会接受 AI 生成的文档吗？**
A：关键看质量。如果 AI 生成的文档经过你的审核和优化，质量不降反升，团队会接受。

## 最后一句

AI 不是来替代产品经理的，是来**替代产品经理的重复劳动**的。

会用 AI 的 PM，把时间花在：
- 深度思考而不是重复写文档
- 用户洞察而不是整理数据
- 战略决策而不是做 PPT

**记住**：AI 是工具，你是决策者。用对了，效率提升 40%；用错了，产出质量下降。
