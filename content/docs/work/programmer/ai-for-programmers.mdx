## GitHub Copilot 实战技巧

### 基础配置与快捷键

安装 Copilot 插件后，以下快捷键能大幅提升效率：

- **Tab**：接受补全建议
- **Alt + ]/[**：跳转到下一个/上一个建议
- **Ctrl + Enter**：查看完整建议列表
- **Ctrl + Shift + P**：手动触发补全

### 场景一：智能代码补全 ⌨️

Copilot 最强大的能力是理解上下文后自动补全。试试在注释中描述你想要的功能：

```python
# 用户注册函数：验证邮箱格式、密码强度，创建用户记录
def register_user(email: str, password: str) -> dict:
    # Copilot 会自动生成完整实现
```

**提示**：写好注释是关键！越清晰的注释，补全质量越高。

### 场景二：快速生成测试用例 🧪

```python
# 基于现有函数，Copilot 可自动生成测试
# 输入: 编写针对 register_user 的单元测试
# 使用 pytest 框架，包含正常和异常情况

import pytest
from app import register_user

class TestRegisterUser:
    def test_valid_registration(self):
        """正常注册流程"""
        result = register_user("test@example.com", "SecurePass123")
        assert result["status"] == "success"
    
    def test_invalid_email(self):
        """无效邮箱格式"""
        with pytest.raises(ValueError):
            register_user("invalid-email", "Pass123")
```

### 场景三：代码重构与优化 🔧

选中需要重构的代码，发送 prompt：

```markdown
请帮我重构以下函数，要求：
1. 添加类型提示
2. 优化错误处理
3. 提升可读性
4. 保持原有功能不变

def process_data(d):
    result = []
    for i in d:
        if i > 0:
            result.append(i * 2)
    return result
```

## AI 辅助调试实战 🐛

### Bug 定位三板斧

```markdown
# 1. 提供错误上下文
这个函数在用户提交表单时偶尔报错：
[粘贴代码]

# 2. 提供错误日志
Error: Cannot read property 'value' of undefined
at processForm (form.js:45)

# 3. AI 分析
请分析可能的原因，给出修复方案
```

### 常见 Bug 类型与 AI 解决方案

| Bug 类型 | 典型表现 | AI 解决思路 |
|---------|---------|------------|
| 空指针异常 | `Cannot read property` | 添加空值检查，提供防御性编程建议 |
| 异步问题 | 数据顺序错乱 | 推荐 async/await 或 Promise 方案 |
| 类型错误 | Type mismatch | 添加 TypeScript 类型或类型守卫 |
| 内存泄漏 | 性能逐渐下降 | 分析引用关系，识别未清理的订阅 |

## Prompt Engineering for Developers 💡

```markdown

# Context:

3. [可维护性需求]

# Output Format:
- 代码实现
- 单元测试
- 使用示例

# Constraints:
- 遵循 [代码规范/框架约定]
- 不引入额外依赖
```

### 调试 Prompt 模板

```markdown
我遇到了 [问题描述]：
```

```

请按以下步骤分析：
1. 可能的原因有哪些？
2. 最可能的原因是哪个？为什么？
3. 如何复现这个问题？
4. 提供修复代码
5. 如何避免类似问题？
```

## 最佳实践与注意事项 ⚠️

### ✅ 推荐做法

1. **始终审查 AI 生成的代码**
   - AI 也会犯错，重要逻辑必须人工审核
   
2. **提供充分的上下文**
   - 项目结构、依赖版本、编码规范都很有帮助

3. **迭代优化 Prompt**
   - 不满意结果时，补充细节重新提问

4. **建立团队 AI 使用规范**
   - 统一 Prompt 模板
   - 设立代码审查环节

### ❌ 避免踩坑

| 误区 | 说明 |
|-----|------|
| 盲目复制粘贴 | AI 代码可能有安全漏洞 |
| 忽略许可证 | 部分代码可能存在许可证风险 |
| 过度依赖 | 保持独立思考和问题解决能力 |
| 泄露敏感信息 | 不要在 Prompt 中包含密钥、密码 |
